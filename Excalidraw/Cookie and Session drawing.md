# Classic (Disk)
1) Classic (Disk)

For example, in PHP if you create a session, 
by default all of the information will be written to /tmp directory

This approach has its own problems
Too much disk IO and as a result req, res speed will become slow
Is this info too important to keep writing to disk? - No
It will be deleted once the user presses logout

If there are 2 webapps and a reverse proxy to spread the requests,
you will need to sync the session with the other webapp too

Therefore, disk approach is not the correct approach in modern times

2) DB

Session info is stored in a DB, when the user with cookie comes,
query is made to the DB sessions table, and the DB responds by
giving out user info.

In this case there won't be any sync problems as the other web app
uses the same DB

It has its own problems too. Performance ^9SWr4ZH8

3) Redis

Redis is an in-memory storage. In keeps info in key-value pairs
Because it is in-memory, it is quite fast

At worst, if something happens to Redis, you can redirect the 
user to /login with HTTP 302 code ^Eirp4M9y

4) Cookie-based Session

This approach is very different from other approaches
In this approach, the session info is kept on client side ^ZdWSKaOV
